# Data Availability Map

This file maps the data and inputs in the paper *fix pimd/langevin: An
Efficient Implementation of Path Integral Molecular Dynamics in LAMMPS* to the
files in this repository.

## Figures 1-5: LAMMPS and i-PI Correctness

These figures compare liquid-water PIMD trajectories, thermodynamic
distributions, and RDFs between `fix pimd/langevin` and i-PI.

- Plotting notebook: `Figures/FIG1-5/lmp-ipi-correctness.ipynb`
- Plotted PDFs: `Figures/FIG1-5/kcv.pdf`, `temperature.pdf`, `pcv.pdf`,
  `density.pdf`, and `rdf_liquid_water.pdf`
- LAMMPS commit-`b1ef9f` 5 ns provenance used for the final paper comparison:
  `Figures/FIG1-5/lmp.lan.npt.b1ef9f/`
- Five-run i-PI/DeePMD `dp_ipi` provenance used for the final paper comparison:
  `Figures/FIG1-5/ipi.lan.npt.dp_ipi.5simulations/`
- RDF arrays used by the notebook: `g_1_1.npy`, `g_1_2.npy`, and `g_2_2.npy`
  in both run directories
- DP model used by these figure inputs: `Figures/FIG1-5/graph.pb`

The model is also provided with the reusable liquid-water example at
`Examples/LiquidWater/graph.pb`.

The `ipi.lan.npt.dp_ipi.5simulations/` archive contains the intended five
independent i-PI simulations used for the paper comparison. Each subdirectory
`0` through `4` includes `input.xml`, `run.slurm`, `mps-wrapper.sh`,
`water.json`, the corresponding `init*.xyz`, and `simulation.out`. Large
per-bead `simulation.pos_*.xyz` trajectories and restart files are intentionally
not archived.

The plotting notebook was updated from the working notebook that used the 5 ns
paper inputs. Stale 1 ns/dev comparison cells were removed from the repository
copy, so it now runs against only `lmp.lan.npt.b1ef9f/` and
`ipi.lan.npt.dp_ipi.5simulations/`.

The `lmp.lan.npt.b1ef9f/` archive contains the corresponding final LAMMPS
correctness run from commit `b1ef9f`, including `conf.lmp`, `in.lammps`,
`run.slurm`, `mps-wrapper.sh`, `rdf.slurm`, per-bead `log.0` through `log.31`,
and the RDF arrays. The old 1 ns `lmp.lan.npt/` job is intentionally not
archived. Trajectory dumps and restart files are intentionally not archived.

## DPLR Liquid-Water Example

The DPLR example discussed in the paper is available under
`Examples/LiquidWater-DPLR/`.

- LAMMPS input and configuration files: `Examples/LiquidWater-DPLR/LAMMPS/`
- DPLR model: `Examples/LiquidWater-DPLR/scan0_lr.pb`
- Deep Wannier model: `Examples/LiquidWater-DPLR/wfc_lr.pb`
- Example logs backing the performance values quoted in the text:
  `Examples/LiquidWater-DPLR/LAMMPS/log.0`,
  `log.0.8gpus`, and `log.0.h2o1024`

## Figures 7-11: Benchmarks and Scaling

These figures are generated by `Figures/FIG7-11/benchmark.ipynb`.

- 512 H2O benchmark: `Figures/FIG7-11/512_h2o/output.dat` and `i-pi.dat`
- 4096 H2O benchmark: `Figures/FIG7-11/4096_h2o/output.dat` and `i-pi.dat`
- One-GPU-per-bead scaling: `Figures/FIG7-11/32_GPU/output.dat` and
  `Figures/FIG7-11/1bead_1GPU/output.dat`
- 175616 H2O strong scaling:
  `Figures/FIG7-11/175616_h2o_ring_repair/output.dat` and
  `Figures/FIG7-11/175616_h2o_1bead/output.dat`
- Weak scaling:
  `Figures/FIG7-11/175616_h2o_weak_scaling_ring_repair/output.dat` and
  `Figures/FIG7-11/175616_h2o_1bead_weak_scaling/output.dat`
- Rendered figures: `512H2O.pdf`, `4096H2O.pdf`, `1GPU_per_Bead.pdf`,
  `175616H2O.pdf`, and `weak_scaling.pdf`

The compact plotted benchmark tables are present in the repo. For Figures 7 and
8, the LAMMPS job provenance is also archived under
`Figures/FIG7-11/512_h2o/` and `Figures/FIG7-11/4096_h2o/`: each GPU-count
subdirectory contains the relevant `in.lammps`, `run.slurm`, `mps-wrapper.sh`,
and short `log` file, and each case includes `extract.py`, `make_task.sh`, and
`template_task/`.

For Figures 9-11, the LAMMPS job provenance is archived under
`Figures/FIG7-11/32_GPU/`, `Figures/FIG7-11/1bead_1GPU/`,
`Figures/FIG7-11/175616_h2o_ring_repair/`,
`Figures/FIG7-11/175616_h2o_1bead/`,
`Figures/FIG7-11/175616_h2o_weak_scaling_ring_repair/`, and
`Figures/FIG7-11/175616_h2o_1bead_weak_scaling/`. These directories include
the relevant `in.lammps`, `run.slurm`, `mps-wrapper.sh` when used, short `log`
files, `extract.py`, `make_task.sh`, and `template_task/`.

Large trajectories, dumps, restart files, and generated working directories are
intentionally not archived.
